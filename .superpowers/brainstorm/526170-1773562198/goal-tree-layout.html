Goal Tree Mobile Layout

How should the goal tree work on mobile? Pick the interaction model that feels right.

A

Flat Card List

Each goal is a tappable card with indent levels showing hierarchy. Expand a card to see tracker widget, progress slider, and actions. Like Todoist/Things.

FITNESS
Run Marathon
Run 5K under 25min
Lose 10kg

Pros

  • Everything visible at once
  • Familiar pattern
  • Fast scanning

Cons

  • Gets crowded with many goals
  • Little room for tracker widgets
B

Drill-Down Navigation Recommended

Start at domains, tap into a domain to see goals, tap a goal to get its full workspace: tracker widget, sub-goals, progress, actions. Breadcrumb nav at top.

Domains > FITNESS > Run Marathon

Pros

  • Clean, focused UI per goal
  • Room for tracker widgets
  • Scales to any depth

Cons

  • Can't see full tree at once
  • Extra taps to navigate
C

Collapsible Tree

Indented list with expand/collapse per node. Tap a node to open a bottom sheet with actions and tracker widget. Like VS Code's file explorer.

▼ FITNESS 47%
▼ Run Marathon 35%
▶ Run 5K under 25min
▶ Complete training plan
▶ Lose 10kg 60%
▶ CAREER 20%
Bottom sheet appears on tap with tracker + actions

Pros

  • Full hierarchy visible
  • Familiar tree pattern
  • Quick expand/collapse

Cons

  • Gets messy with deep nesting
  • Bottom sheet context-switch
