# Session: March 11, 2026 - Axiom Load Balancing & Deep Diagnostics

## Summary of Changes
- **Axiom (AI Coach) Load Balancing:**
  - **Randomized Entry:** Updated `AICoachingService.ts` to pick a **random starting key** from the pool for every new request. This prevents the system from always exhausting the first few keys while leaving the last ones (like Key #6) unused.
  - **Explicit Key Status:** Improved the debug reporter to show the status of **every unique key** in the pool. Identical error messages are no longer collapsed, so you can see exactly which keys hit 429 (Quota) vs 404 (Disabled API).
  - **Dud Key Deprioritization:** If a key returns a 403 (Forbidden) or 401 (Invalid), the system now breaks from that key's model loop immediately to find a working key faster.
- **Provider Resilience:**
  - Integrated **DeepSeek** as the primary provider (if `DEEPSEEK_API_KEY` is present), falling back to the 7-key Gemini pool.
  - Unified error reporting across all providers.
